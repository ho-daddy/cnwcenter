 3. 핵심 기능 요구사항 (Implementation Tasks)T1. 공통 스크래퍼 프레임워크 (Common Scraper)사이트별로 개별 코드를 짜지 말고, BaseScraper 클래스를 상속받아 Selector와 Parser만 정의하는 구조로 설계할 것.Library: axios + cheerio (정적), 필요한 경우 playwright (동적) 사용.Error Handling: 사이트 구조 변경에 대비한 예외 처리 및 로깅 필수.T2. 키워드 기반 필터링 (Smart Filtering)수집된 데이터 중 다음 키워드가 포함된 항목에 가중치를 부여할 것.키워드: 사망, 중대재해, 구속, 판결, 산안법 개정, 위험성평가, 고용노동부 점검.T3. Claude API 연동 브리핑 생성수집된 원문 데이터를 Claude API에 전달하여 다음 형식의 마크다운 리포트를 생성할 것.[오늘의 3대 핵심 이슈]: 가장 중요한 뉴스 3건 요약.[상세 브리핑]: 사고/정책/판례 카테고리별 요약.T4. 데이터 관리link_hash를 사용하여 중복 수집을 방지할 것.결과물은 reports/briefing_YYYYMMDD.md 형태로 저장하거나 DB(SQLite 등)에 적재할 것.4. 클로드코드를 위한 실행 가이드 (Step-by-Step)환경 구성: npm init 및 typescript, cheerio, axios, dotenv 설치.프로토타입 작성: 우선 '고용노동부'와 '매일노동뉴스' 2곳을 대상으로 하는 스크래퍼 코드를 작성하고 동작 확인.확장: 나머지 소스(법원 판결, 안전신문 등)를 추가하고 ScraperFactory 패턴 적용.LLM 통합: 수집된 텍스트를 정제하여 Claude에게 요약을 요청하는 AnalysisService 구현.스케줄링: node-cron을 이용해 매일 아침 08:30에 실행되도록 설정.
